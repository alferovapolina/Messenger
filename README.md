Что нужно сделать, чтобы заработал мессенджер:

1. Скачать PostgreSQL с официального сайта. Во время установки задать имя пользователя "postgres" и пароль "postgres". С другими данными представленный в данном репозитории код работать не будет, придется его немного изменить. Кроме того, забытый пароль невохможно восстановить, поможет только полный снос PostgrSQL и переустановка.
2. Скачать проект messenger из данного репозитория.
3. Выполнить настройку PostgeSQL на платформе Dochost (проделать только лишь этапы "Создание диска", "Создание файла конфигурации", "Создание и настройка контейнера") с помощью этого гайда: https://docs.dockhost.ru/recipes/db/postgresql
4. Захостить проект messenger с помощью Docker Desktop, Selectel и Dochost, а также создать домен. Для этого проделать видеогайд для крестиков-ноликов из прошлого семестра.
5. Готово!

Мессенджер работает следующим образом. На первой странице требуется ввести имя пользователя. Если пользователя нет в базе данных, то создается новый с соответствующим именем. Далее появится список чатов пользователя. Здесь можно создать новый чат, либо ввести имя уже существующего чата, в котором состоят другие пользователя. Так или иначе, для каждого пользователя каждый чат нужно иницииализировать первый раз путем ввода его имени. На странице чата отображается история сообщений, можно писать новые сообщения.

В случае, если мессенджер работает некорректно после перехода по ссылке домена, вызываем в браузере на странице мессенджера меню клавишой F12 и отрабатываем все выплывающие в логах ошибки. В случае проблем с базами данных помогает прописать в консоли Package Manage Console внутри Visual Studio команду Update-Database, после чего заново опубликовать проект и перезапустить контейнер в Dochost.
